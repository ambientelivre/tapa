Mind the structure

The name of the template you add here is the name of the folder you zipped. This mean that you should pack a template by typing something like $ zip -r my-template.zip ./my-template. So beware of sending a template with the same name of another preexistent one.

If you're sending a fresh new template, it's also good to have in mind the basic template structure:

  • assets/ folder Everything that needs to be publicly accessible needs to be in that folder. Have in mind that you're providing a public webpage and that the images, scripts and css files your template is gonna need, must be into this folder.
  • template-config.json This file has all the configurations that belong to the template and must bring the following structure: 
    {
    /* The relative path to the screenshot of your template */
    "thumbnailImageUrl" : "assets/img/pentaho-screenshot.png",

    /* resources to load into PUC - can be an empty array if none */
    "externalResources" : [
        {
            "type": "js",
            "context": "mantle",
            "url": "assets/js/mantle-context.js"
        }
    ],


    /* Configurable text tags in your template - can be an empty array if none */
    "textTags" : [
        {
            "tag": "STATIC_RIGHT_HEADER_TEXT",
            "value": "Business Intelligence Portal"
        }
    ]
}
  • index.html This is the file you need to put your Login interface into.
    IMPORTANT: When referencing your template files, use the variable {{TAPA_TEMPLATE_ROOT_URL}}. Example: 
    <link href="{{TAPA_TEMPLATE_ROOT_URL}}/assets/css/styles.css" rel="stylesheet" type="text/css">
