Insert your company logo

We're delivering along with our tapa-default template a way to inject your company logo into your User Console toolbar.

If you're just a user that also wants your logo there, be our guest and replace that logo with yours, by clicking the edit button mode_edit choosing the right image to replace.

But if you're a little more curious on how it's done, pease take a seat. We've just used one of the Pentaho's plugins built-in features that is the ability to inject Javascript and CSS files into different contexts. For placing that logo over there, we've just injected one CSS telling where's the image and styling the element and one JS file that creates the element where we want. That's it.

Replace Pentaho's welcome page

With the same strategy that we've used for placing your logo on PUCs toolbar, we can also give your home screen a new interface. If you want to enable this example, just make sure to choose our tapa-defailt and include into its injections, the file welcome-redirect.js.

If you're a developer and wants to customize your welcome-page, place your simple html into your template folder and edit a welcome-redirect.js like file to do your redirection. This javascript only does it right for the CE Edition, though it's also possible to do this tweak on the Enterprise Edition.

Inject CSS/Javascript into Mantle

We've mentioned that injecting such types of files is a built-in feature of the Pentaho's plugin platform. We've just made that a little easier here.

This feature is more helpful for developers and guys that know what they're doing. So if you're one of those guys, i bet you know what it does. To choose what files are going to be injected just click the edit button mode_edit and choose the right ones from within your template structure.

Develop your own pagkage

Our login pages templates are processed using Pebble. These templates are server-side processed and we did it so all the presentation layer is separated from the server-side code.

If you want to develop your own login template page, you can get started copying one of ours. But if you want to do it from scratch, just make sure to do at least a login form that submits a POST to j_spring_security_check passing j_username and j_password. That's it.

And as a server-side processed template, we've also provided a list of variables that our original PUCLogin.jsp page used, so you can use these in your template with the notation {{ VARIABLE_NAME }}. Here is the list: 

	  TAPA_TITLE          Messages.getInstance().getString("UI.PUC.TITLE")
	  TAPA_LOCALE         ESAPI.encoder().encodeForHTMLAttribute(request.getLocale().toString())
	  TAPA_LOGIN_TITLE    Messages.getInstance().getString("UI.PUC.LOGIN.TITLE")
	  TAPA_USERNAME       Messages.getInstance().getString("UI.PUC.LOGIN.USERNAME")
	  TAPA_PASSWORD       Messages.getInstance().getString("UI.PUC.LOGIN.PASSWORD")
	  TAPA_LOGIN          Messages.getInstance().getString("UI.PUC.LOGIN.LOGIN")
	  TAPA_EVAL_LOGIN     Messages.getInstance().getString("UI.PUC.LOGIN.EVAL_LOGIN")
	  TAPA_ADMIN_USER     Messages.getInstance().getString("UI.PUC.LOGIN.ADMIN_USER")
	  TAPA_BUSINESS_USER  Messages.getInstance().getString("UI.PUC.LOGIN.BUSINESS_USER")
	  TAPA_GO             Messages.getInstance().getString("UI.PUC.LOGIN.GO")
	  TAPA_COPYRIGHT      Messages.getInstance().getString("UI.PUC.LOGIN.COPYRIGHT", String.valueOf(year))
	  TAPA_ERROR_CAPTION  Messages.getInstance().getString("UI.PUC.LOGIN.ERROR.CAPTION")
	  TAPA_LOGIN_ERROR    Messages.getInstance().getString("UI.PUC.LOGIN.ERROR")
	  TAPA_LOGIN_OK       Messages.getInstance().getString("UI.PUC.LOGIN.OK")
	  TAPA_REQUESTED_URL  ESAPI.encoder().encodeForJavaScript(requestedURL)
	  TAPA_SHOWUSERS      showUsers
	  TAPA_LOGGEDIN       loggedIn
	  TAPA_TEMPLATE_ROOT_URL /* Path to your template so you only use relative paths on html*/

These variables, along with the ones you define in your user-interface, are passed down to the template on the server side.

IMPORTANT: When referencing your template files, use the variable {{TAPA_TEMPLATE_ROOT_URL}}. Example: 

<link href="{{TAPA_TEMPLATE_ROOT_URL}}/assets/css/styles.css" rel="stylesheet" type="text/css">
Doing this way you keep your template name-independent.

If you're developing an Angularjs app for your login page and you want your angular template variables to be rendered with the curly brackets, make sure to write them as {{ng ANGULAR_VAR }}. Mind the ng. Click here for more information.
